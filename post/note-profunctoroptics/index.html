profunctor optics: higher-order datatype-generic representation of data accessors.

access a component of type A within a composite structure of type S.
component A can be replaced by one of type B, yielding a new composite of type T.

notation:
- $A \times B$ for $(A,B)$.
- $A + B$ for $Either\ A\ B$.
- subscript of a type class to hint which instance is being used:
$fmap_{List}$ is the List instance of fmap.
- $f : A \simeq B : g$ for $f : A \to B$, $g : B \to A$ form an isomorphism.

2.2 Categorical Prerequisites

category $\mathbb{C }$ consists of:
- collection of objects: $\vert \mathbb{C} \vert$;
- set of arrows from $A$ to $B$ for each $(A,B)$: $\mathbb{C}(A,B)$;
- identity arrow;
- composition for arrows.
laws:
- composition is associative;
- identity arrow as neutral elements for composition.

category $\mathbb{Set}$:
objects are sets,
homset $\mathbb{Set}(A,B)$ consisting of the set of total functions from A to B.

category $\mathbb{Pre} (A,\leq )$:
objects are elements of A,
homset $\mathbb{Pre} (A,\leq )(a,b)$ consisting of a single element when $a\leq b$ and empty otherwise.

catrgory $\mathbb{Mon } (M,\oplus ,e)$:
single object * ,
sole homset $\mathbb{Mon } (M,\oplus ,e)(* ,* )$,
identity arrow is the neutral element $e$,
composition is given by $\oplus $.

categories are structured entities.
functors are structure-preserving mappings between categories.

homfunctor:
homset $\mathbb{C} (A,B)$ is a set of arrows, so $\mathbb{C} (A,-)$ is a functor from $\mathbb{C}$ to $\mathbb{Set} $,
taking $B$ (object of $\mathbb{C} $) to $\mathbb{C} (A,B)$ (object of $\mathbb{Set} $),
taking $f \in \mathbb{C} (B,C)$ (arrows of $\mathbb{C} $) to $(f \circ ) \in \mathbb{Set} (\mathbb{C}(A,B),\mathbb{C}(A,C))$.

$\mathbb{C} (-,B)$ is a functor from $\mathbb{C} ^{op}$ to $\mathbb{Set}$.

functor $F:\mathbb{Mon} (M,\oplus ,e) \to \mathbb{Set} $
taking sole object * to some set $S \in \vert \mathbb{Set} \vert$,
taking each arrow $m:* \to *$ to a function $S \to S$.
($S \to S$ respects composition ($\oplus) $ and neutral element ($e$),
thus $S \to S$ is a left action of the monoid $(M,\oplus,e)$ on S.)

homfunctor $\mathbb{Mon} (M,\oplus ,e) (*,-) : \mathbb{Mon} (M,\oplus ,e) \to \mathbb{Set}$
taking sole object * to the set $M$,
taking each arrow $m:* \to * $ to the function $(m\oplus -): M \to M$,
it is the left action of $(M,\oplus , e)$ on $M$ itself.

functors are structured entities.
natural transformations are structure-preserving mappings between functors.

two functors: $F,G:\mathbb{C} \to \mathbb{D}$,
natural transformation $\phi: F \to G$ is
a family of arrows in $\mathbb{D}$, indexed by objects in $\mathbb{C}$.
an arrow $\phi _A \in \mathbb{D} (F(A), G(A))$, for each object $A \in \vert \mathbb{C} \vert$,
satisfying the naturality condition:
$\phi _B \circ F(f) = G (f) \circ \phi _A$, for each $f \in \mathbb{C} (A,B)$.

for any two categories $\mathbb{C}, \mathbb{D}$,
functor category $\lbrack \mathbb{C}, \mathbb{D} \rbrack $
objects $\vert \lbrack \mathbb{C}, \mathbb{D} \rbrack \vert$, the functors from $\mathbb{C}$ to $\mathbb{D}$,
arrows $\lbrack \mathbb{C}, \mathbb{D} \rbrack (F,G)$, the natural transformations $F \to G$ (between $F,G \in \vert \lbrack \mathbb{C}, \mathbb{D} \rbrack \vert$).

yoneda lemma: for locally samll category $\mathbb{C}$, there is a bijection
$\lceil - \rceil ^{A,F} : \lbrack \mathbb{C}, \mathbb{Set} \rbrack (\mathbb{C}(A,-),F) \simeq F(A) : \lfloor - \rfloor ^{A,F}$

$\lceil - \rceil ^{A,F}$,
take a natural transformation $\phi : \mathbb{C} (A,-) \to F$, yield an element of the set $F(A)$.
applying a component of the family $\phi$ to identity arrow,
$\phi _A : \mathbb{C} (A,A) \to F(A)$, $id_A \in \mathbb{C}(A,A)$,
$\lceil \phi \rceil ^{A,F} = \phi _A (id_A)$.

$\lfloor - \rfloor ^{A,F}$,
take an element $x$ of the set $F(A)$, yield a natural transformation $\mathbb{C} (A,-) \to F$
object $B \in \mathbb{C}$, $\phi_B$ is an arrow in $\mathbb{Set}$, a function from $\mathbb{C}(A,B)$ to $F(B)$.
giving an $f\in \mathbb{C} (A,B)$, map $f$ over $x$ using the functorial action of $F$,
$\lfloor x \rfloor _B ^{A,F} (f) = F(f)(x)$.

$\lceil - \rceil$ direction, application to the identity arrow
$\lfloor - \rfloor$ direction, use of the functorial action

dual version of the yoneda lemma:
$\lbrack \mathbb{C} ^{op}, \mathbb{Set} \rbrack (\mathbb{C} (-,B),F) \simeq F(B)$

whole homfunctor $\mathbb{C} (-,=)$ is a functor $\mathbb{C}^{op} \times \mathbb{C} \to \mathbb {Set}$.
curry the product category. yield a functor $\mathbb{C} ^{op} \to \lbrack \mathbb{C} ,\mathbb{Set} \rbrack$, written ‘$H^{\bullet }$’

Theorem 2.1 (Yoneda Embedding).
the functor $H^{\bullet } : \mathbb{C} ^{op} \to \lbrack \mathbb{C}, \mathbb{Set} \rbrack$ is full and faithful, and injective on objects.

functor is full: surjective on each homset
functor is faithful: injective on each homset

$\lbrack \mathbb{C}, \mathbb{Set} \lbrack (\mathbb{C} ((A,-),(B,-)) \simeq \mathbb{C} (B,A)$

class Functor f where
fmap :: (a -> b) -> (f a -> f b)
-- law: fmap (g . h) = fmap g . fmap h
-- law: fmap id = id

class Profunctor p where
dimap :: (c -> a) -> (b -> d) -> p a b -> p c d
-- law: dimap (f' . g) (g . g') = dimap f g . dimap f' g'
-- law: dimap id id = id

class Profunctor p => Cartesian p where
second :: p a b -> p (c × a) (c × b)
-- assoc : (A × B) × C ≃ A × (B × C) : assoc'
-- unit : 1 × A ≃ A : lunit'
-- law: dimap assoc assoc' (second_C . second_D h)= second_C×D h
-- law: dimap lunit lunit' h = second_1 h

class Profunctor p => Cocartesian p where
right :: p a b -> p (c + a) (c + b)
-- coassoc : (A + B) + C ≃ A + (B + C) : coassoc'
-- lzero : 0 + A ≃ A : lzero'
-- law: dimap coassoc coassoc' (right_C . right_D h) = right_C+D h
-- law: dimap lzero lzero' h = right_0 h

type AdapterP a b s t = forall p. Profunctor p => p a b -> p s t

type LensP a b s t = forall p. Cartesian p => p a b -> p s t

type PrismP a b s t = forall p. Cocartesian p => p a b -> p s t
instance Profunctor (->) where
dimap f g h = g . h . f 

instance Cartesian (->) where 
second f (c , a) = (c , f a)

instance Cocartesian (->) where
right f (Left c) = Left c
right f (Right a) = Right (f a)

flattenP :: AdapterP (a × b × c) (a' × b' × c') ((a × b) × c) ((a' × b') × c')
flattenP h = dimap f t h where
f ((a , b) , c) = (a , b , c)
t(a , b , c) = ((a , b) , c)

sndLensP :: LensP a b (c × a) (c × b)
sndLensP h = second h

theP :: PrismP a b (Maybe a) (Maybe b)
theP h = dimap m2s s2m (right h) where
          m2s Nothing = Left ()
          m2s (Just x) = Right c
          s2m (Left ()) = Nothing
          s2m (Right x) = Just x

lenses and prisms are divergent generalizations of adapters.
adapters: access onto a value via a change of representation.

‘transformer’ type constructor $P$.
for profunctor $P$, a transformer of type $P\ A\ B$ consumes $A$ values and produce $B$ values.

profunctor $P$ is cartesian (strong) if it is coherent with products.

there is an operator second that for any type $C$,
lifts a transformer of type $P\ A\ B$, to a transformer of type $P\ (C × A)\ (C × B)$,
that acts on pairs, transforming $A$s into $B$s but passing $C$s around unchanged.

profunctor $P$ is co-cartesian (choice) if it is coherent with the structure of sums.

Examples of Yoneda

indirect inequality
$(b \leq a) \Leftrightarrow (\forall c.(a \leq c) \Rightarrow (b \leq c))$

full and faithful functors both preserve and reflect isomorphisms.

any functor preserves isomorphisms.
i.e. if $f$ and $g$ form an isomorphism, so do $F(f)$ and $G(f)$.
$F(f) \circ F(g) = id$
$\Leftrightarrow [[ F\ is\ a\ functor ]]$
$F(f \circ g) = F (id)$
$\Leftarrow [[ Leibniz ]]$
$f \circ g = id$

for a full (surjective on arrows) and faithful (injective on arrows) functor $F$,
arrow $f$ forms an isomorphism if and only if $F(f)$ forms an isomorphism.

yoneda embedding provides a full and faithful functor, therefore
$\mathbb{C}(B,-) \simeq \mathbb{C}(A,-) \Leftrightarrow (A \simeq B) \Leftrightarrow \mathbb{C}(-,A) \simeq \mathbb{C}(-,B)$

if one regards $\mathbb{C} (-,A) (U) = \mathbb{C} (U,A)$ as ‘$A$ viewed from $U$’, then this states that two objects $A$, $B$ are the same if and only if they look the same from all viewpoints $U$.

currying: a function from a product type is equivalent to a function to an exponential.
$\mathbb{Set} (A \times C, B) \simeq \mathbb{Set} (A,B^C)$

a function from a sum type is equivalent to a pair of functions with a common target.
$\mathbb{Set} (A+B,C) \simeq \mathbb{Set}(A,C) \times \mathbb{Set}(B,C)$

$\mathbb{Set} ((A + B) \times C , X)$
$\simeq [[ currying ]]$
$\mathbb{Set} (A+B,X^C)$
$\simeq [[ sum ]]$
$\mathbb{Set} (A,X^C) \times \mathbb{Set} (B,X^C)$
$\simeq [[ currying ]]$
$\mathbb{Set} (A \times C, X) \times \mathbb{Set} (B \times C, X)$
$\simeq [[ sum ]]$
$\mathbb{Set} ((A\times C) + (B \times C), X)$

$(A + B) \times C \simeq (A \times C) + (B \times C)$
